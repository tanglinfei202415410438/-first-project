# 图书库存管理系统

## 项目目标
开发一个简易的图书库存管理系统，用于帮助书店或图书馆管理员管理图书的入库、销售、库存和供应商信息。

## 功能需求
### 图书信息管理
可以添加、查看、修改和删除图书信息。
每本图书包括以下字段：
- 书名
- 作者
- 出版社
- 出版日期
- 库存数量
### 供应商管理
记录图书供应商的信息：
- 供应商名称（制作商）
- 联系方式
- 类型（如：出版社、分销商）
- 状态（是否合作中）
### 采购与入库管理
记录采购订单：
- 订单日期
- 图书名称
- 数量
- 状态（如：已下单、已入库）
- 更新入库后自动增加库存数量。
### 销售管理
记录每笔销售：
- 销售日期
- 图书名称
- 销售数量
- 所属门店
- 销售后自动减少库存数量。
### 库存查询
- 可按书名、作者、出版社等查询图书库存。
- 显示库存数量与入库日期。

## 用户角色
- 管理员：负责所有信息的录入、修改和删除。

## 非功能需求
- 使用简单的图形界面（如 Java Swing 或 Python Tkinter）。
- 数据存储在本地文件中（如 CSV 或 SQLite 数据库）。
- 系统响应迅速，操作简单易懂。

## 开发建议
- 使用面向对象编程（OOP）设计图书、订单、供应商等类。
- 可先实现控制台版本，再逐步加入图形界面。
- 注意数据一致性，如销售时检查库存是否充足。
